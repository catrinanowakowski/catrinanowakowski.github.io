And if someone else happens to be interested in it, that's great too!

How I framed this project

When I start a project, one of the first things I consider is how many times am I going to have to perform a function, or need this feature. Usually, I then start to construct a function, and I make it as MODULAR as I can.

But what was interesting with this project, is that my goal was different from what I usually do. Usually, I am constructing an analysis or data visulization, where I pass data in, and get either slightly different data or a graphical component out. For this project, I am making a multiple static pages for another human to navigate, but also a file structure and map that I need to be able to scale and mold as my needs and know how grow.

First things first - pick something overly complicated for your index page.

I need a navigation pane, and two photos of the ocean that stretch my index.html page that make two fancy looking sections because I cannot just start with a simple background apparently. While, this lead to me breaking my page multiple times early on, it was useful to start with a challenging task (that I still havn't mastered, one photo is wider still just slightly wider than the other as I am writing this), because it meant I had to overcome a few barriers early before I had fleshed out my page infrastructure.
